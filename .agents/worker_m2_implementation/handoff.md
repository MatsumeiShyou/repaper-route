## 2. Logic Chain

1. **PeriodicJobImporter**: By replacing Nth weekday matching logic to explicitly check both `dayKey` and `${dayKey}${nth}` in Object format, and exact match or `${dayKey}${nth}` in Array format, we ensure correct job scheduling for specific weeks.
2. **nativeFetch**: By iterating over `localStorage.length` and using `localStorage.key(i)` before falling back to `Object.keys`, token extraction is highly robust across standard and custom mock storage environments.
3. **serialization**:
   - Explicit null-checks for numbers and parsing string boolean values `"true"`/`"false"` prevents erroneous zero-coercion and incorrect double negation.
   - Using the case-insensitive regex flag (`/i`) in `normalizeDays` captures all cases of day keys.
   - In `cleansePurgedFields`, returning `Date/RegExp/Map/Set` immediately skips cloning. Incorporating a `WeakSet` detects cyclic reference recursion and returns the reference to safely break stack overflows.
4. **sortUtils**: Added checks for `a == null` and `b == null` at the beginning of `universalSort` to prevent `TypeError` crashes, mapped `NaN` to the end stably, and added `Date` object type guards in `isValidDate`.
5. **MasterDataLayout**: Refactored `PointAccessSection` to fetch raw permissions and map them client-side in-memory using `drivers` and `vehicles` state arrays, fully resolving Postgrest view join bad requests. Wrapped primary key values in `encodeURIComponent` to prevent URL layout corruption.
6. **MasterDataContext**: Added `Array.isArray(dRes.data)` checks to avoid mapping over unexpected payloads.
7. **E2E & Test Infra**: Corrected `testDir` paths in `playwright.config.ts`, added a stub mock `VLMClient` with standard verification method, and set Vitest environment to `jsdom` in `nativeFetch.test.ts`.
8. **Type Safety & compile/test verification**: Refactored all `any` declarations in the M2 files to `unknown`, typed parameter schemas, and confirmed compiling and testing pass successfully.
