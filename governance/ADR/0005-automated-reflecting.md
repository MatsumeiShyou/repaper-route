# ADR 0005: 自動反映プロトコル (Reflecting-by-Default) の導入

## ステータス
承認済み / 実装中

## 背景 (State)
2026年3月8日のタスク実行において、検証（closure_gate）が成功したにもかかわらず、最終的な `git push` が AI の記憶・優先順位から漏れ、反映が未完了のまま終了するという事態が発生した。
現在の統治構造は「検証成功」と「反映」が物理的に結合されておらず、最終工程が AI の自発的な意志に依存していることが原因である。また、長時間稼働するバックグラウンドプロセスがファイルをロックし、反映を妨げるケースも確認された。

## 意思決定 (Decision)
「検証 (Verification)」と「反映 (Reflection)」をアトミックな単一の物理シーケンスとして結合する **Governance-as-Code v6.0** を導入する。

1.  **物理連鎖**: `closure_gate.js` に `--reflect` フラグを導入し、テスト合格直後に `git commit` & `git push` を自動実行する。
2.  **事前同期**: push 直前の競合を避けるため、反映シーケンス内で `git pull --rebase` を強制する。
3.  **プロセスパージ**: 反映直前に `npm test` 等のゾンビプロセスを強制終了させ、ファイルロックを物理的に排除する。

## 理由 (Reason)
- AI の認知負荷に依存せず、スクリプトの exit code 0 以外ではタスクを完遂させない物理的制約により、反映漏れを構造的に不可能にするため。
- Windows 環境特有のファイルロックリスクを自動で排除し、反映成功率を極大化するため。

## リスク評価 (Risk)
- **失敗率予測**: 1% 以下（物理連鎖による安定性向上）
- **緩和策**: スクリプト内の各ステップでエラーチェックを厳格に行い、一つでも失敗した場合は全プロセスを安全に中断する。
