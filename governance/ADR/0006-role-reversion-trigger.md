# ADR-0006: Role Reversion Trigger (役割自動遷移の義務化)

- **Status**: Accepted
- **Date**: 2026-03-08
- **Deciders**: Antigravity, User

## 1. Context (背景)

これまでの統治構造では、Analyzer（設計者）と Executor（執行者）の役割分離が規定されていたが、ユーザーから「計画を立てろ」等の設計指示を受けた際、AIがそのまま Executor モードで推測実装を継続してしまうリスクがあった。
これにより、設計フェーズ（PLANNING）をスキップした不完全な実装が行われる可能性があり、100pt の健全性を維持するための課題となっていた。

## 2. Decision (判断)

ユーザーからの設計指示を検知した際、即座に **ANALYZER 役割（PLANNING モード）** へ復帰することを、憲法第1層 §C-4 において「絶対的義務」として定義する。

また、これを単なる精神論に留めないため、以下の物理的制約を導入する。
1. **憲法上の絶対律化**: 違反を「統治ロック」の対象とする。
2. **物理門番（Locker）の強化**: 設計指示がありながら設計証跡（ADR/Planの更新）がない実装反映を、`closure_gate.js` で物理的に遮断する。

## 3. Consequences (結果)

### Positive
- ユーザーの意図（設計の重視）がシステム的に保証される。
- 推測実装による手戻りや品質低下が構造的に防止される。
- 役割の切り替えが Task Boundary (PLANNING → EXECUTION) によって物理的に証明される。

### Negative / Risks
- 低リスクな軽微な修正において、役割の切り替えが冗長に感じられる可能性がある。（これについては T1/T2 ティアの適切な運用でカバーする）
