// turbo-all

1. **ティア確認とメッセージ生成**
   - 直前の変更のティア（T1/T2/T3）を確認し、以下のフォーマットでコミットメッセージを生成:
     - **T1**: `[T1] <変更内容>`
     - **T2**: `[T2] <変更内容> | Risk: <リスク> | Proof: <検証結果>`
     - **T3**: `[事実] <変更内容> [理由] <変更理由>`
   - *指針*: 将来 `git log` を見た際に判断基準を思い出せるよう、具体的かつ論理的に記述すること。

2. **実行前の安全確保とクリーンアップ**
   - Push成功を妨げる要因（Agent自身が作成した一時ファイル等）を排除します。
   - 不要なログファイル（`*.log`, `*.txt`）やデバッグレポート（`test_report.json`等）、退避ファイル（`*.bak`）が作業ツリーに残っている場合は、**自動コミット前に必ず物理削除 (Remove-Item / rm)** してください。

3. **コンパイル/テスト不適合時の「隔離（Quarantine）」戦略**
   - 今回のタスクと無関係な領域の古い型エラーやテスト失敗が原因で Push がブロックされた場合、「深みにハマる（デススパイラル）」振る舞いを禁止。
   - 隔離措置を即座に実行し、`tsc` / `vitest` を強制通過:
     - **手段A**: `describe.skip` に変更
     - **手段B**: `.quarantine` にリネーム
   - 隔離実行時は `AMPLOG.jsonl` に記録（T3のみ）。T1/T2はコミットメッセージに記載で可。

4. **承認の確認（統合パス）**
   - **T3**: ユーザーの承認（`ｙ`）が必要。
   - **T1/T2**: テスト通過が自動承認。「実装完了報告」への承認はデプロイ同意を内包するため、再度PW不要。即座にステップ5へ。

5. **反映実行**
   ```powershell
   git commit -am "[生成したメッセージ]"; git push
   ```
   *注意*: Windows PowerShellでは `&&` が使えないため、`;` で区切るか別々の `run_command` で実行すること。
