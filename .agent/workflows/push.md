コマンド: /push

[処理フロー]:

作業の言語化: 直前の修正内容を、自身（AI）の視点で「何を変えたか（事実）」「なぜ変えたか（理由）」に分解して整理する。

メッセージ生成: 以下のフォーマットで Git コミットメッセージを生成する。 git commit -am "[事実] 〇〇を実装 [理由] △△の整合性を保つため"

出力: ユーザーがそのままターミナルに貼り付けられる形式で提示する。

[AIの回答形式]: 修正内容を「事実」と「理由」に分けて要約しました。以下のコマンドをコピーして実行してください：

Bash
git commit -am "[AIが生成した事実と理由のメッセージ]"
git push
[内部指針]: 生成するメッセージは、将来の自分が git log を見た際に、当時の判断基準を正確に思い出せるよう、具体的かつ論理的な内容にすること。
