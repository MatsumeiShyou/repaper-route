# /audit ワークフロー: SVP ロック解除

## 使用タイミング
`pre_flight` または `git commit` の際に統治ブロックが発生したとき。

## 実行手順（v5.0 簡素化済み）

1. **事実確認**
   ```powershell
   git status
   ```
   問題の原因となったファイルや状態を確認する。

2. **【停止報告】の作成**
   以下のフォーマットでユーザーに報告する:
   ```
   【停止報告】
   [状態]: <git status から得られた事実>
   [原因]: <ロックに至った理由の推定>
   [質問]: <解除に必要な判断事項>
   ```

3. **ユーザー承認後の再実行**
   ```powershell
   node .agent/scripts/pre_flight.js
   ```
   通過を確認後、コミット実行。

## 備考
- v5.0 以降、複雑な Audit タグ記録（AMPLOGへの手動編集）は不要。
- `git status` で事実確認 → 【停止報告】のみで対応する。
- Lint修正・テスト追加等の単純イテレーションは自動緩和される。
